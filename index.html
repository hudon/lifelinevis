Tag Propagation Over Time

A tree node represents a process thread. An edge between two nodes represents communication between them. The parent is the process that sends the tag, and the child is the receiving process. The tree nodes are placed in time buckets based on the time that the tag was received at. Thus, the time indicated at each node is the time at which a tag was received, not sent.

Here, real data is used after assigning a tag ("2") to firefox threads and waiting for a few seconds. One tag name was changed to "0" to show an example legend.

Aside from visual enhancements, a worthwhile extension to the graph could be focusing on certain time periods (e.g. if there is one hour's worth of data, view only what happened during the 30th minute).

Tag Propagation Graph

Here, every process thread is a node. Edges are formed between nodes that have communicated with eachother.

The different tags are shown as edges with different colors. These colors are the same as the ones used in the tree above for the same tags. Use the checkboxes below to show only certain tags.

Variations on this graph could include representing multiple connections through edge thickness as opposed to multiple edges and adding communication direction.

Thread Communication Frequency Matrix

The processes are listed on the vertical axis (sent a tag) and on the horizontal axis (received a tag). The colored squares in the graph represent the amount of communication between the source and the target process. Hovering over a square highlights the processes involved in the communication.
NOTE: Data used here differs from the above two.

Total Interations Per Process

The x-axis contains a list of all the process threads that either sent or recieved tags. The y-axis measures the total number of interactions for the tread. The left stacked bar measures the number of tags sent by the process. The right stacked bar measures the number of tags recieved by the process. Different colors within the bar represent different tags.
